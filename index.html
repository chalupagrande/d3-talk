Data Visualization and Design Crash Course

Users of data visualization (paraphrasing from Edward Tufte) are executing particular analytical tasks such as making comparisons or determining causality. The design principle of the data visualization should support the analytical task -- nothing more and certainly nothing less.

  • Show the data.
  • Let the user think about the substance (data) rather than the methodology, technology or design.
  • Never distort or contort the data.
  • Present many numbers in a small space.
  • Make large data sets coherent.
  • Encourage the eye to compare different pieces of data.
  • Reveal the data at several levels of detail.
  • Server a reasonably clear purpose: description, exploration, tabulation.
  • Be closely integrated with the statistical and verbal description of a data set.

Should I use D3 or a different Charting Library?

  • Do I have a good familiarity with Web APIs, ES6 and OOP?
  • Do I have lots of time to build a chart from scratch?
  • Do I have a Visual Designer to help with Typography, color, accessibility?
  • Do I need a custom chart with features to be added later? Will the design change?
  • Is my dataset huge or does my chart needs to be fast on the web?
  • Do I need to integrate my chart with a front end framework?

D3's General Update Pattern

              
  let group = svg.append('g').attr('class','bubble-group')
  group.selectAll('.bubbles')
            .data(data)        // }
            .enter()           // } <= the important bit
            .append('circle')  // }
            .attrs({
              ...
            })

What d3 does, is it creates an empty selection. You can use .empty() to find that out. It then binds the array of data to that selection using .data(data). .enter() tells D3: "for every piece of data I have that doesn't have a corresponding element from the selection, create one. Finally, .append('circle') tells D3 what kind of element to append.

Exit Pattern

If your selection wasn't empty, lets say you had 20 elements in it, and only 10 points of data. You could use: 

              
group.selectAll('.bubbles')
            .data(data)
            .enter()   // <=
            .append('circle')
            .attrs({
              ...
            })
            .exit()    // <= the opposite of .enter()
            .remove()

This will get rid of the unnecessary elements on the page.

Dynamic Attributes

If you pass an attribute a function instead of a static value, D3 will pass that function three arguments.

  • The data for that respective element (ie {r: value})
  • The index of that respective element in the data set
  • The array of raw elements from the selection
  • The this value is bound to the element itself
